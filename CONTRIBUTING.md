# Contributing Guidelines

Thanks for your interest in contributing! This list aims to be a high-quality, curated collection for the Hermes Agent ecosystem.

## How to Suggest a Resource

**[Open an issue](https://github.com/muhajirdev/awesome-hermes-agent/issues/new)** with:

- **Name** and URL
- **Author** (GitHub handle or name)
- **Description** of functionality
- **Category** (see README for options)
- **Why it belongs here** — what makes it valuable?

Please don't submit PRs directly to add resources. Maintainers review and add items to ensure consistent quality and tone.

## Quality Criteria

Resources should be:

1. **Relevant** — Directly tied to Hermes Agent or agentskills.io
2. **Documented** — Clear README explaining purpose and usage
3. **Working** — Functional (or clearly marked experimental/WIP)
4. **Unique** — Not duplicating existing entries
5. **Open** — Publicly available source code

## Maturity Labels

Entries are tagged **production**, **beta**, or **experimental** based on docs quality, maintenance, and adoption. Think a label needs updating? [Open an issue](https://github.com/muhajirdev/awesome-hermes-agent/issues/new) — we'd rather fix labels than waste your time on something unfinished.

## List Improvements

For suggestions about categories, formatting, or structure, [open an issue](https://github.com/muhajirdev/awesome-hermes-agent/issues/new) describing your idea.

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). Participation means agreeing to its terms.
